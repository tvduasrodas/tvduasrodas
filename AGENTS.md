# Regra de commit e push

1. Faça o commit diretamente, sem usar o plugin Computer Use.
2. Se não for possível fazer o commit diretamente, abra um novo chat com o plugin Computer Use para fazer o commit e o push.
3. Se o commit direto funcionar, tente fazer o push remoto diretamente.
4. Somente se o push remoto direto falhar, por exemplo com erro 443, abra um novo chat com o plugin Computer Use para fazer o push manualmente pelo GitHub Desktop.

Ao usar o chat alternativo com o plugin Computer Use, envie exatamente esta mensagem, sem acrescentar ou remover texto:

abrir Github Desktop app e fazer commit de tarefas pendentes em caso nao houver necessidate de fazer commit fazer push.
